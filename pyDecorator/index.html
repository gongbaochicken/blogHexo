Python Decorator

Posted on 2018-02-07 | In Python | | Page View

The decorator in Python is a function that takes another function and EXTENDS the behavior of the latter function without explicitly modifying it. Some other programming languages or frameworks has the similar idea, for example, Angular by Google has the decorator in UI component too.

An interesting thing is that many people complains about the name of “Decorator” in python, because they think this is not what describes in GoF book, and “decorator” should be used in compiler or design pattern area. Yep, but now it is still named as “decorator”.

Let’s jump out of the code scenario, and give an example of Tony Stark, the Iron Man. Tony is the core, and while armor can be different for various purpose. He can wear “Mark 16” (which is shining and cool he thinks) just for night club, but he can also switch back to “Hulkbuster” to fight against . Suits or armors can change, but the core stay focused.

Also suit can borrowed by other guys, like James Rhodes(Iron Patriot) and Captain American.



While Decorator acts like the similar way, providing some extra functionality, it does has such benefits:

  1. Reduce code replication by extracting and reusing code, because those part can be reused multiple times in multiple places when they are needed.
  2. Extracting the parts unrelated to main function(like trace, log, time record, etc), so the original function can focus on the main purpose.

For example, you want to make some time records or log printout for different modules/methods, then the timer or log component can be written as decorator. Another example I see before is that make configuration with different parameters before running python tests, then the configuration method is a decorator in that code.

Decorator with arguments

More element and amazing thing is that the decorator can passed extra parameters, like other normal functions. With these pass-in parameters, we can easily change its logic and make it more powerful. For some example, by passing “error” or “info”, it can do different things when record time or trace , this is very useful when we need to change internal operations with same decorator.
